# 项目架构文档

## 1. 项目概述

阿尔兹海默症诊断系统是一个基于深度学习的多模态医学影像诊断平台，集成了MRI影像、临床特征、生活方式和分子生物标志物数据，通过先进的深度学习模型实现阿尔兹海默症的早期诊断和风险评估。

## 2. 系统架构

### 2.1 整体架构

```

├── models/                 # 模型定义目录

├── utils/                  # 工具函数目录

```

### 2.2 核心模块

#### 2.2.1 模型模块 (models/)

- **MultiModalADModel**: 多模态阿尔兹海默症诊断模型
  - 3D-CNN影像编码器：提取MRI影像特征
  - 临床特征编码器：处理临床数据
  - 生活方式特征编码器：处理生活方式数据
  - 分子特征编码器：处理分子生物标志物数据
  - 注意力融合机制：融合多模态特征
  - 双任务输出：分类任务（诊断）和回归任务（风险预测）

#### 2.2.2 数据处理模块 (src/train_model.py)

- **ADNIDataset**: 数据集类，支持数据加载和预处理
  - 数据归一化和标准化
  - 数据增强（噪声添加、对比度调整、亮度调整、随机翻转）
  - 真实数据加载和合成数据生成

#### 2.2.3 数据下载模块 (utils/data_downloader.py)

- **DataDownloader**: 数据下载器类
  - 支持多个公开数据集下载（ADNI、OASIS、MIRIAD、NACC、ADRC、UCSD、Harvard AD）
  - 样本数据生成功能
  - 目录结构创建和管理

#### 2.2.4 配置模块 (config/config.py)

- **CONFIG**: 系统配置字典
  - 输出类别定义（CN、EMCI、LMCI、AD）
  - 风险指标定义
  - 数据模态定义

## 3. 数据流

### 3.1 训练数据流

```
原始数据 → 数据预处理 → 数据加载 → 模型训练 → 模型保存 → 模型评估
```

### 3.2 推理数据流

```
输入数据 → 数据预处理 → 模型推理 → 结果生成 → 报告输出
```

## 4. 技术栈

- **编程语言**: Python 3.11+
- **深度学习框架**: PyTorch 2.0+
- **医学影像处理**: nibabel, numpy
- **数据处理**: pandas, scikit-learn
- **可视化**: matplotlib, seaborn
- **Web框架**: Flask (用于前端展示)
- **测试框架**: unittest

## 5. 关键技术

### 5.1 多模态融合

采用注意力机制进行多模态特征融合，能够自动学习不同模态的重要性权重，提高模型的泛化能力和诊断准确率。

### 5.2 3D-CNN特征提取

使用深度3D卷积神经网络提取MRI影像的空间特征，能够捕捉大脑结构的细微变化。

### 5.3 双任务学习

同时进行分类任务（阿尔兹海默症诊断）和回归任务（12个月进展风险预测），提高模型的综合性能。

### 5.4 数据增强

实现多种数据增强技术，包括噪声添加、对比度调整、亮度调整和随机翻转，提高模型的鲁棒性。

### 5.5 早停机制

使用早停策略防止模型过拟合，保存最佳模型状态。

## 6. 扩展性设计

### 6.1 模型扩展

系统设计支持添加新的模型架构和算法，只需在models目录中添加新的模型类。

### 6.2 数据扩展

支持添加新的数据集和数据模态，通过配置文件和数据加载器的扩展实现。

### 6.3 功能扩展

系统采用模块化设计，新功能可以通过添加新的模块实现，不影响现有功能。

## 7. 部署考虑

### 7.1 硬件要求

- **训练**: GPU (至少8GB显存)
- **推理**: CPU或GPU均可

### 7.2 环境配置

- Python 3.11+
- PyTorch 2.0+
- CUDA 11.7+ (GPU训练)

### 7.3 部署方式

支持本地部署和云端部署，可通过Docker容器化部署。
